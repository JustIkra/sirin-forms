# Smoke-сценарии UI

Этот файл — источник правды по user-scenarios, которые агент прогоняет через Playwright.
Агент использует его при команде **«я задеплоил новую фичу — обнови smoke и прогони заново»** (см. `scripts/qa-update-smoke.py`).

## Активные сценарии

### 1. `01-forecast-generate.spec.ts` — генерация недельного прогноза
**Что проверяет:** пользователь открывает главную, видит форму «Недельный прогноз», указывает сегодняшнюю дату в MSK, жмёт «Сгенерировать» → появляется таблица с блюдами и метаданные.
**Скриншоты:** `01-forecast-initial`, `01-forecast-generated`.

### 2. `02-forecast-filter-sort.spec.ts` — фильтры и сортировка
**Что проверяет:** после генерации прогноза пользователь включает фильтр «ML», сортирует по имени блюда. Каждая видимая строка помечена `data-method="ml"`.
**Скриншоты:** `02-forecast-filtered`.

### 3. `03-plan-fact-discrepancy.spec.ts` — план-факт прошлой недели
**Что проверяет:** пользователь указывает понедельник прошлой недели → прогноз генерируется, план-факт подтягивается, кнопка ИИ-анализа расхождений становится видимой.
**Скриншоты:** `03-plan-fact-loaded`.

### 4. `04-inventory-load.spec.ts` — остатки и закупка
**Что проверяет:** переход во вкладку «Остатки», ввод даты, загрузка таблицы. Переключение табов на «Нужно закупить».
**Скриншоты:** `04-inventory-all`, `04-inventory-to-buy`.

### 5. `05-navigation.spec.ts` — навигация
**Что проверяет:** переключение между пунктами сайдбара «Прогноз» ↔ «Остатки». Заголовки страниц меняются, форма остаётся валидной.
**Скриншоты:** `05-nav-forecast`, `05-nav-inventory`.

### 6. `06-daily-analysis.spec.ts` — дневной анализ
**Что проверяет:** переход на вкладку «Анализ на день», форма видна, попытка построить прогноз на дату из прошлого не крашит UI.
**Скриншоты:** `06-daily-initial`, `06-daily-result`.

## Как добавить новый сценарий

1. Опиши пользовательский путь текстом в формате выше (Что проверяет / Скриншоты) **в этом файле**.
2. Создай `NN-<название>.spec.ts` с использованием общих `data-testid` из `src/frontend/src/components/**`.
3. Скриншоты — через `shoot(page, 'имя')` из `helpers.ts` (сохранятся в `reports/screenshots/`).
4. Для дат — только `todayMsk()` / `pastMondayMsk(N)` из `helpers.ts` (MSK), никаких `new Date()`.
5. Прогони: `cd src/frontend && npx playwright test NN-...`.

## Конвенции

- Один `test()` = один сценарий. Не делай multi-step в одном test через хаки — разбивай.
- Используй `data-testid` вместо xpath/css-классов. Если testid нет — попроси, чтобы его добавили.
- Жди элементы через `expect(locator).toBeVisible({ timeout: 45_000 })`, не через `waitForTimeout`.
- Баги UI → сделай скриншот (Playwright делает автоматически при падении: `screenshot: 'only-on-failure'`).
